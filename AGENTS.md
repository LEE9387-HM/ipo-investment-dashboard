# AGENTS

이 파일은 `ipo-investment-dashboard` 프로젝트 내에서 활동하는 모든 에이전트가 준수해야 할 구체적인 규칙을 정의합니다.

## 1. 구현 가이드라인
- **Python**: Type Hinting을 적극적으로 사용하고, `logging` 모듈을 사용하여 실행 과정을 기록합니다.
- **Data**: CSV 파일을 읽고 쓸 때는 `pandas` 라이브러리를 선호합니다.
- **HTML/CSS**: 외부 프레임워크(Tailwind 등)를 사용하지 않고 Vanilla CSS로 작성하며, `index.css`에 중앙 집중식 디자인 시스템을 구축합니다.
- **GitHub Actions**: 모든 워크플로우는 `data/` 폴더의 CSV를 갱신하고 자동으로 커밋/푸시하도록 설계합니다.

## 2. 보안 가이드라인
- `.env.local`에 정의된 변수명을 코드에서 사용하되, 기본값을 비워두어 실수가 없도록 합니다.
- `print()`를 통한 데이터 출력 시 민감한 정보가 포함되지 않도록 주의합니다.

## 3. 검증 가이드라인
- 새로운 스크립트를 작성하면 반드시 샘플 데이터로 로컬 실행 테스트를 완료한 후 보고합니다.
- 대시보드 수정 시 브라우저 렌더링 결과(스크린샷 등)를 확인합니다.

## 4. 특이사항
- 공모주 데이터 수집 시 DART의 '증권신고서'와 '투자설명서'를 주 소스로 활용합니다.
- Gemini API 호출 시 `google-generativeai` 라이브러리를 사용합니다.
